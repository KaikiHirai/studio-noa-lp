# 写真の差し込みガイド（STUDIO NOA LP）

実写を入れる準備はできています。**元写真を `photos-src/` に置いて 1 コマンド**で最適化・差し替えできます。

```bash
# 1) 元写真を photos-src/ に「スロット名」で置く（拡張子は自由: jpg/png/webp/heic 等）
#    例: photos-src/hero.jpg, photos-src/t1.png, photos-src/room.jpeg
# 2) 最適化（public/img/ に <スロット名>.jpg を生成）
python3 scripts/optimize-images.py
# 3) 反映
npm run build          # 本番。もしくは npm run dev で確認
```

- スクリプトが**表示比率へ自動でカバー切り抜き＋圧縮**します（被写体は中央寄せ推奨）。
- 一部だけ差し替えても OK（置いたスロットだけ更新、残りはプレースホルダー据え置き）。
- `photos-src/` は公開ビルド `out/` に含まれないため、原本（高解像度）を置いたままで安全です。
- 画像を替えたら **`src/lib/content.ts` の各 `alt`** を写真内容に合わせて更新してください（アクセシビリティ／SEO）。

## スロット一覧

| スロット名 | 使用箇所 | 比率 | 書き出し | 目安容量 | 読み込み | 被写体の狙い |
|---|---|---|---|---|---|---|
| `hero` | ファーストビュー主役 | **4:5** | 900×1125 | ≤70KB | **eager / 最優先** | 明るい個室 or 女性トレーナーの柔らかい表情。第一印象を決める最重要カット |
| `room` | 理由①完全個室 | 4:3 | 1000×750 | ≤90KB | lazy | 扉で仕切られた1対1の個室。清潔感 |
| `trainer-work` | 理由②女性トレーナー | 4:3 | 1000×750 | ≤90KB | lazy | 女性トレーナーが会員に寄り添って指導する様子 |
| `meal` | 理由③食事指導 | 4:3 | 1000×750 | ≤90KB | lazy | 彩りのよい無理のない食事 |
| `t1`〜`t4` | トレーナー紹介4名 | **5:6** | 600×720 | ≤70KB | lazy | 上半身ポートレート。`t1`=森かおり(代表) → `t2`高橋 → `t3`石井 → `t4`中村 の順 |
| `voice1`〜`voice3` | お客様の声アバター | 1:1 | 200×200 | ≤20KB | lazy | 実名NGなら後ろ姿・手元・雰囲気カットでも可 |
| `ogp` | SNSシェア画像 | 1.91:1 | 1200×630 | ≤130KB | — | ブランドカット。文字入れするなら中央を空けておくと安全 |

## 重要な注意

- **`hero` の容量は死守**：ファーストビュー画像はモバイル LCP 要素です。ここが重いと Lighthouse Performance（現在 97）が落ちます。目安 **70KB 以内**。スクリプトが超過時に警告します。
- **顔出しの効果**：女性専用ジムでは「女性トレーナーのみ・安心」を写真で証明できると離脱が大きく下がります。特に `hero` と `t1〜t4` の顔出しは強く推奨。
- 現在入っているのは配置確認用の抽象プレースホルダーです。差し替え前でもレイアウトは崩れません。
- WebP を使う場合はこのスクリプトを使わず `public/img/` に直接置き、`src/lib/content.ts` の `src` 拡張子を変更してください（`hero` は素の `<img>` なので同様にパス変更）。
